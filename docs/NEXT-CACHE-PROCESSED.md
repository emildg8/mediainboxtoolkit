# Кэш «уже обработано» (следующий шаг)

Идея: локальный JSON (например `LOGS/processed-inbox-state.json`) с хэшами путей или `(размер, mtime)` исходников, чтобы при повторном скане не предлагать те же переносы и не трогать стабильную библиотеку. Включение через политику `state.enabled`. Не реализовано в 0.1.3 — только контракт для следующей итерации.
