# Классификация контента (дорожная карта 2.x)

Цель: по **смыслу** (не только по шаблону имени) отнести файл к типу и к целевой ветке библиотеки.

| ID | Тип | Сигналы (план) |
|----|-----|----------------|
| 2.1 | Фильм | Нет SxxEyy; TMDB movie; длительность ~90–180 мин (ffprobe); один файл в релизе |
| 2.2 | Сериал (эпизод) | SxxEyy; TMDB TV; сезон в базе |
| 2.3 | Фильм, относящийся к сериалу | Special/movie в TMDB к TV id; ключевые слова pilot/featurette |
| 2.4 | Мультфильм | TMDB animation + movie; жанр/студия; отсутствие эпизодного кода |
| 2.5 | Мультсериал | Как 2.2 + мульт-контекст (студия, жанр, путь) |
| 2.6 | Мультфильм к сериалу | Связь TMDB / коллекция |
| 2.7 | Аниме полнометражка | Япония/origin; movie; нет TV episode структуры |
| 2.8 | Аниме-сериал | Эпизодный код + anime |
| 2.9 | Аниме-сериал «ветка» | Связанные шоу TMDB |
| 2.10 | OVA к основному аниме | OVA/спешл; parent TV в TMDB |

Текущая реализация (`MediaInboxToolkit.ContentKinds.ps1`): **эвристики v0** (эпизодный код, ключевые слова anime/мульт). Дальше: жанры TMDB, `keywords`, длительность, коллекции, локальный **rules store** (см. OFFLINE-METADATA).
